Default Parameters, Rest Parameters, Named Parameters
parámetros

En ES5, que habíamos variando las formas de manejar funciones que necesitan los valores por defecto , argumentos indefinidos , y los parámetros con nombre . Con ES6, podemos lograr todo esto y mucho más utilizando una sintaxis más concisa.
Default Parameters
parámetros por defecto


function addTwoNumbers(x, y) {
    x = x || 0;
    y = y || 0;
    return x + y;
}

En ES6, simplemente podemos suministrar valores por defecto para los parámetros en una función: 


function addTwoNumbers(x=0, y=0) {
    return x + y;
}

addTwoNumbers(2, 4); // 6
addTwoNumbers(2); // 2
addTwoNumbers(); // 0

Rest Parameters
Parámetros de descanso; en ES5, manejamos un número indefinido de argumentos, así:


function logArguments() {
    for (var i=0; i < arguments.length; i++) {
        console.log(arguments[i]);
    }
}

Utilizando el operador rest, podemos pasar en una cantidad indefinida de argumentos:


function logArguments(...args) {
    for (let arg of args) {
        console.log(arg);
    }
}

Named Parameters
Parámetros con nombre; uno de los patrones en la ES5 para manejar los parámetros con nombre era utilizar las opciones de objeto de modelo, adoptado de jQuery.


function initializeCanvas(options) {
    var height = options.height || 600;
    var width  = options.width  || 400;
    var lineStroke = options.lineStroke || 'black';
}

Podemos conseguir la misma funcionalidad utilizando desestructuración como un parámetro formal a una función:


function initializeCanvas(
    { height=600, width=400, lineStroke='black'}) {
        // Use variables height, width, lineStroke here
    }

Si queremos que todo el valor opcional, podemos hacerlo por desestructurar un objeto vacío:


function initializeCanvas(
    { height=600, width=400, lineStroke='black'} = {}) {
        // ...
    }
